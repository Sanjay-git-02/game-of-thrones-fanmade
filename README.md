## Quick Start (run locally)

Prerequisites:

- Node.js 18+ (or latest LTS)
- Git

Steps to run:

1. Clone the repo:

```bash
git clone https://github.com/<owner>/<repo>.git
cd Game-of-Thrones
```

2. Install dependencies:

```bash
npm install
```

3. Start dev server (hot reload):

```bash
npm run dev
```

Open http://localhost:5174/ in your browser.

4. Build for production:

```bash
npm run build
npm run preview
```

## Assets

This repo keeps site assets in `public/assets/` so they are served automatically. If you intentionally exclude large binary assets from the Git history, use one of these approaches so users can still obtain them after cloning:

- Commit small assets directly to `public/assets/` (recommended for images/GIFs under a few MB).
- Use Git LFS for large binaries (install `git lfs`, track filetypes, push LFS objects).
- Host assets on a CDN or GitHub Releases and reference remote URLs in code.
- Provide a `scripts/fetch-assets.js` that downloads assets at `postinstall` time (not included by default).

If assets are missing after clone, check `public/assets/` and update references in code (paths use `/assets/<filename>`).

## Notes and troubleshooting

- If `node_modules/` is present in your working tree, remove it before committing (add it to `.gitignore`).
- If Windows reports locked files when deleting `node_modules`, close running editors/servers or restart your machine, then run:

```powershell
rd /s /q node_modules
rd /s /q dist
```

- For build errors referencing missing plugins, run `npm install` to restore dependencies.

## 📁 Project Structure

- `index.html` — app entry
- `package.json` — scripts & dependencies
- `public/` — static files served as-is
- `public/assets/` — images, GIFs, icons used by the app
- `src/` — application source
- `src/components/` — UI components (including `Dragons.jsx`)
- `src/main.jsx` — React entry
- `src/index.css`, `src/App.css` — styles
